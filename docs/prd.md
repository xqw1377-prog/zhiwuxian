# WUXIAN PRD（产品需求文档）

## 1. 背景与问题

传统学习产品的共性问题：

- 用打卡/惩罚驱动，导致用户“掉队即羞耻”
- 线性课程结构对个体差异适配不足
- 长视频学习成本高：冗余段落占用大量时间
- 过程不可解释：只给“完成率/分数”，缺少证据链与可复盘资产

WUXIAN 的目标是：让用户在真实状态（疲惫、波动、碎片化）下仍能推进，并把“时间压缩能力”产品化。

## 2. 目标（Goals）

- 把“目标推进”做成低摩擦系统：用户掉队时系统静默重算，而不是催促与惩罚
- 把“长视频学习”做成结构化路径：可定位关键节点、可触发虫洞跳跃
- 把“学习过程”变成资产：雷达卡/证书/海报可分享
- 把“算力消耗”变成计费闭环：钱包、订单、解锁、token 校验

## 3. 非目标（Non‑Goals）

- 不做 B 端后台与机构运营能力
- 不做内容生产平台
- 不承诺“全程离线不落库”（当前以本机 SQLite 为主，仍会存遥测/事件）

## 4. 目标用户（Personas）

- 自学者：愿意长期自驱，但执行稳定性差，需要反惩罚托底
- 极客学习者：追求效率与“系统外挂感”，愿意为时间压缩付费
- 轻度焦虑学习者：不愿面对催促型产品，希望“温和但不放弃”的陪伴

## 5. 核心体验（用户旅程）

### 5.1 冷启动

1) 打开产品（Cockpit 或 Dashboard）  
2) 自动创建/恢复会话与钱包状态  
3) 用户输入一句目标或一个视频链接

### 5.2 量子投喂（意图驱动）

- 输入包含 URL → 触发“视频折叠”
- 输入表达疲惫/卡住 → 触发“重算”
- 输入为聊天/倾诉 → 触发“陪聊”

### 5.3 长视频折叠与虫洞

1) 同化视频：提取知识节点并写入图谱  
2) 指针路由：在节点上进行“3 分钟切片定位”  
3) 达标触发：虫洞跃迁（跳过冗余节点）

### 5.4 战利品与分享

- 免费：雷达卡与海报（SVG）
- 付费：证书解锁（生成 token 化分享 URL）

## 6. 功能需求（MVP → 可迭代）

### 6.1 Goal OS（目标操作系统）

- 目标拆解：输入 goal + 天数，输出任务与陪伴语
- 重路由：完成/太难 → 调整斜率并生成下一日任务
- 夜巡：可选的静默 detect（未来扩展通知/回访）

### 6.2 Quantum（量子意图）

- 意图解析：稳定输出结构化 JSON
- 会话推进：pulse/complete 维持“航线状态”
- 星卡：生成可视化战利品预览（未来可扩展生成 PNG）

### 6.2.1 路径 A：逆向目标与时间斜率账本

- 逆向推演：输入目的地/现状/倒计时，输出认知包分母与初始分子
- 倒计时进度条：在驾驶舱顶部持续显示，作为“以终为始”的硬核仪表
- 冷启动恢复：从本地账本读取并恢复 UI 状态

### 6.3 Video（视频折叠）

- 同化：基于视频管道/启发式生成知识节点
- 课程图谱：按课程查询节点
- 指针路由：courseId+timestamp → 当前节点/跳转建议

### 6.4 Telemetry（遥测）

- ingest：事件上报
- aggregate：按窗口聚合 metrics + evidence
- 报告：雷达卡与海报生成

### 6.5 Billing/Wallet/Payment（商业闭环）

- 钱包：credits、warp_minutes、订阅 tier、每日/月重置
- 商品目录：warp pack / subscription / credits / report unlock
- 订单：创建订单、confirm（simulate）、webhook（live）
- 报告解锁：下单 → 支付成功 → 生成 shareUrl（token）

## 7. 指标（Success Metrics）

- 激活：首次投喂完成率、首次折叠成功率
- 留存：D1/D7 投喂与重路由复访率
- 体验：同化成功率、虫洞触发率、证书生成与分享率
- 收入：付费转化（pack/订阅/证书解锁）、ARPPU

## 8. 定价与套餐（当前实现口径）

- Warp Power：免费额度 + 10h 包 + 无限月卡
- 订阅：Growth/Pro（能力解锁与无限）
- 证书：单次解锁
- Credits：小额算力包

## 9. 风险与约束

- 隐私叙事与实现的差距：需要持续推进脱敏与最小化上传策略
- 图谱版本堆积：需要后续 GC/归档策略
- 真实支付接入：需要更严格验签、重放防护与对账
